\section{DFS}

\subsection{Домашнее задание}
\begin{enumerate}
  \item
    Пусть дано дерево $T = \langle V, E \rangle$. Для каждого ребра вычислить, сколько
    простых путей проходит через него. $\O(|V| + |E|)$.

    Решение:

    \begin{itemize}
      \item Так как нам дано дерево, то между двумя любыми вершинами существует только один простой путь.
      \item При этом каждое ребро разбивает дерево на 2 компоненты связности. Соответственно, в 1-ой компоненте мы можем выбрать, где начинаем путь, а во 2-ой -- где заканчиваем.

      \item Тогда для дерева на $|V|$ вершинах количество простых путей, проходящих через какое-либо ребро, будет равно $S \cdot (|V| - S)$, где
      \newline $S$ -- размер одной компоненты связности,
      \newline $(|V| - S)$ -- размер другой компоненты связности

      \item Получается, надо для каждого ребра посчитать размер одной из компонент связности, на которые делится дерево при удалении этого ребра.
      
      Сделать это можно, если подвесить дерево за какую-нибудь вершину $r$, обозначив её за корень, а затем посчитать размеры поддеревьев через DFS:
      \begin{enumerate}
        \item Заведём массив $parent[v]$, который для каждой некорневой вершины $v$ хранит её родителя
        \item Заведём массив $size[v]$, который для каждой вершины $v$ хранит размер ее поддерева
        \item Тогда для вершины $v$ размер её поддерева равен $1 + \sum_{u \in children(v)} size(u)$
      \end{enumerate}
      
      \item Тогда для каждого ребра $(v, parent[v])$ количество простых путей вычисляется за $\O(1)$ по формуле $size[v] * (|V| - size[v])$, 
      где $v \neq r$ 

      \item Сложность по времени:
      \begin{itemize}
        \item DFS для подвешивания дерева и заполнения $parent[]$ -- $\O(|V| + |E|)$
        \item Заполнение массива $size[]$ -- $\O(|V|)$
        \item Подсчёт ответа для каждой вершины -- $\O(|V|)$
      \end{itemize}
      
      Итого: $\O(|V| + |E|) + \O(|V|) + \O(|V|) = \O(|V| + |E|)$. Так как в дереве $|E| = |V| - 1$, то итоговая сложность $\O(|V|)$

      \item Сложность по памяти:
      \begin{itemize}
        \item массив $parent[]$ -- $\O(|V|)$. Так как в условии ничего не сказано про способ хранения дерева, то считаем, что потребуется дополнительная память для подвешивания.
        \item массив $size[]$ -- $\O(|V|)$
      \end{itemize}

      Итого: $\O(|V|)$ 

    \end{itemize}

  \item
    Корневое дерево $T$ на $V$ вершинах задается массивом из $V$ элементов. 
    Все вершины пронумерованы. Для каждой вершины в массиве указан его родитель.
    Для корня $r$ значение в массиве равно -1. 
    Требуется определить, как будет выглядеть новое представление 
    дерева, если корень $r$ сменить на корень $q$. Разрешается использовать
    $\O(1)$ дополнительной памяти. Менять массив можно. Время $\O(V)$.

    Решение:

    \begin{itemize}
      \item При смене корня с $r$ на $q$ нужно развернуть рёбра только на пути от $q \leadsto r$.
      Все остальные вершины дерева уже корректно указывают на своих родителей.

      \item Для этого достаточно держать 3 указателя:
      \begin{itemize}
        \item на текущую вершину
        \item на предыдущую вершину на пути $q \leadsto r$ (она станет новым родителем текущей вершины)
        \item на старого родителя текущей вершины (для перемещения по дереву)
      \end{itemize}

      \item На каждом шаге сохраняем старого родителя, меняем родителя на предыдущую вершину пути, переходим дальше.

      Примерный код:
      \begin{verbatim}
      prev = -1
      curr = q
      while curr != -1:
          next = parent[curr]
          parent[curr] = prev
          prev = curr
          curr = next
      \end{verbatim}

      \item Сложность по времени: $\O(|V|)$ -- длина пути от $q \leadsto r$ не превышает $|E|$. Однако нам дано дерево, поэтому $|E| = |V| - 1$

      \item Сложность по памяти: $\O(1)$ -- нужно только 3 фиксированных переменных.
    \end{itemize}


  \item
    За $\O(|V| + |E|)$ найдите в \textbf{неорграфе}:
    \begin{enumerate}
      \item все \textbf{ребра}, через которые проходит любой путь $a \leadsto b$.
      
      Решение:

      \begin{itemize}
        \item Мост -- это ребро, при удалении которого увеличивается число компонент связности графа.

        \item Идея: Ребро $e$ лежит на всех путях $a \leadsto b$ тогда и только тогда, когда $e$ является мостом, разделяющим $a$ и $b$.

        \item Доказательство:
        \begin{itemize}
          \item[$\Rightarrow$] Пусть ребро $e$ лежит на всех путях $a \leadsto b$. Тогда при удалении $e$ не существует пути из $a$ в $b$, значит $a$ и $b$ оказываются в разных компонентах связности. Следовательно, $e$ -- мост.
          
          \item[$\Leftarrow$] Пусть $e$ -- мост, разделяющий $a$ и $b$ (т.е. при удалении $e$ вершины $a$ и $b$ оказываются в разных компонентах). Тогда любой путь $a \leadsto b$ должен пересекать границу между этими компонентами, а единственный способ это сделать -- пройти через ребро $e$.
        \end{itemize}

        \item Алгоритм поиска мостов за $\O(|V| + |E|)$:

        Запустим DFS из вершины $a$. Для каждой вершины $x$ определим:
        \begin{itemize}
          \item $enter(x)$ -- время входа в вершину $x$
          \item $ret(x)$ -- минимальное время входа среди вершин, достижимых из поддерева с корнем $x$:

          $$ret(x) = \min\begin{cases}
            enter(x) \\
            \min_{(u, x) \in up} \{ enter(u) \}, & \text{где } up \text{ -- множество обратных рёбер} \\
            \min_{z \in V} \{ ret(z) \}, & \text{где } z \text{ -- потомок } x
          \end{cases}$$

        \end{itemize}

        Ребро дерева $(x, y)$ (где $y$ -- потомок $x$) является мостом $\Leftrightarrow ret(y) > enter(x)$.
        
        Это означает, что из поддерева $y$ нельзя попасть в $x$ или выше без использования ребра $(x, y)$.

        \item Финальный алгоритм:
        \begin{enumerate}
          \item Запустить DFS из $a$, попутно вычисляя $enter$, $ret$ и сохраняя для каждой вершины её родителя в массиве $parent$
          \item Восстановить путь $a \leadsto b$, идя от $b$ к $a$ по массиву $parent$
          \item Для каждого ребра $(x, y)$ на пути $a \leadsto b$: если $ret(y) > enter(x)$, то это искомое ребро
        \end{enumerate}

        \item Сложность по времени:
        \begin{itemize}
          \item DFS для вычисления $enter$, $ret$, $parent$ -- $\O(|V| + |E|)$
          \item Восстановление пути $a \leadsto b$ по массиву $parent$ -- $\O(|V|)$
          \item Проверка рёбер на пути $a \leadsto b$ -- $\O(|V|)$
        \end{itemize}
        
        Итого: $\O(|V| + |E|)$
        
        \item Сложность по памяти: $\O(|V|)$ -- массивы $enter$, $ret$, $parent$
      \end{itemize}

      \item все \textbf{вершины}, через которые проходит любой путь $a \leadsto b$.
      
      Решение:

      \begin{itemize}

        \item Рассуждение аналогичное: берем путь $a \leadsto b$. Все интересующие нас вершины будут на этом пути.
        
        Если на этом пути есть вершина $x$, которая при удалении разобъет граф на $\geq 2$ компоненты связности,
        то значит $x$ -- это точка сочленения. И она же является вершиной, которая лежит на всех путях $a \leadsto b$.

        \item Поиск точек сочленения за $\O(|V| + |E|)$:

        Вершина $x$ является точкой сочленения $\Leftrightarrow$
        \begin{itemize}
          \item $x$ -- корень дерева и имеет $\geq 2$ детей, или
          \item $x$ -- не корень и существует ребёнок $y$ такой, что $ret(y) \geq enter(x)$
        \end{itemize}

        \item Финальный алгоритм:
        \begin{enumerate}
          \item Найти путь $a \leadsto b$ через DFS
          \item Найти все точки сочленения графа за $\O(|V| + |E|)$
          \item Вернуть точки сочленения, лежащие на пути $a \leadsto b$ (кроме самих $a$ и $b$)
        \end{enumerate}

        \item Сложность по времени: $\O(|V| + |E|)$
      
      \end{itemize}

    \end{enumerate}

  \item
    Дано дерево $T = \langle V, E \rangle$.
    \begin{enumerate}
      \item
        Найдите центроид дерева --- вершину $c$, для которой минимальна \\
        $\sum_{v \in V} dist(v, c)$. $\O(|V|)$.

        Решение:

        \begin{itemize}
          
          \item Возьмем произвольную вершину $x$ и подвесим за нее дерево

          \item Обойдем все дерево через DFS и посчитаем формулу центроида из условия. Получили какое-то значение $d$

          \item Вполне возможно, что это не является ответом на задачу. 
          Однако пересчитывать формулу для каждой вершины неоптимально - будет $\O(|V|^2)$.
          
          \item Вместо этого мы можем сделать следующее:
          \begin{itemize}
            \item Пусть $y$ -- один из потомков $x$

            \item Если мы сдвинемся в вершину $y$, то все расстояния в поддереве $y$ уменьшаться на 1,
            а в остальной части дерева -- увеличатся на 1

            \item Тогда получается формула пересчета: $d - S + (N - S)$, где
            \newline $S$ -- размер поддерева с вершиной $y$,
            \newline $(N - S)$ - размер остальной части дерева.
          \end{itemize}

          \item Таким образом обходим все поддерево, пересчитываем значение по новой формуле и получаем ответ.
          
          \item Сложность по времени: $\O(|V|)$
        
        \end{itemize}

    \end{enumerate}

\end{enumerate}
